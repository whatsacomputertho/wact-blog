Pet projects: as engineers we can all relate to the experience of having way too many of them, and the experience of having that one pet project about which we are particularly passionate.  For me, my highest regarded pet project is to develop a football simulator capable of simulating my and my brother's imaginary football leagues.  In fact, this is the pet project that led me to pursue software as a career path.

As I've grown as a software engineer through my career, I've learned of the value of properly scoping software projects.  I've also learned through personal experience that pet projects are especially vulnerable to improper scoping.  This is partially due to our passion and personal connection to them - we strive for perfection out of our pet projects and as a result they never fully come to fruition.

With this iteration of my football simulator game, I'd like to impose a very well-defined scope to keep myself accountable, and my goals realistic.  I'd like to first approach the problem from the highest level and slowly drill down into the finer-grained details as I progress.

Scope

For the first iteration, I will implement a model that can randomly generate the final score of a football game (sometimes called a "Box Score") given each team's offensive and defensive skill levels.  Additionally,

  • The model should be a good fit against real football data, meaning there will be some initial data science involved before any real development occurs.
  • Once implemented, the model itself and supporting utilities should be packaged as a library atop which a CLI and API can later be implemented.

  • The build & deployment of the full stack should be simple, repeatable, and semi-automated for both local and production deployments.

In this series of blogs I will walk through my progress on each pillar of this project's first iteration, from the data science, to the web development, to the CI/CD, and everything in-between.  Once this first iteration is completed, I will assess my progress and determine how I plan to approach my next iteration in a new blog series.
